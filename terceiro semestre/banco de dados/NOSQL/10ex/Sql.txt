1 - NoSQL (Not Only SQL) é um termo que engloba bancos de dados que não seguem o modelo tradicional dos bancos de dados SQL. NoSQL difere dos bancos de dados SQL em termos de estrutura de dados, escalabilidade, flexibilidade de esquema e modelo de consulta. Nos bancos de dados SQL, os dados são organizados em tabelas com esquemas fixos e relacionamentos definidos, enquanto os bancos de dados NoSQL adotam modelos de dados mais flexíveis, como documentos, grafos, chave-valor ou colunas amplamente distribuídas.

Exemplos de bancos de dados SQL: MySQL, Oracle, PostgreSQL.
Exemplos de bancos de dados NoSQL: MongoDB, Redis, Cassandra, Neo4j.

2 - Os quatro principais tipos de bancos de dados NoSQL mencionados são:

Banco de dados de documentos: Nesse tipo de banco de dados, os dados são armazenados em documentos, geralmente no formato JSON ou BSON. Cada documento é autocontido e pode ter uma estrutura flexível. Exemplo: MongoDB.
Banco de dados de chave-valor: Nesse tipo de banco de dados, os dados são armazenados como pares de chave e valor. Os valores são opacos para o banco de dados e a recuperação de dados é feita por meio da chave. Exemplo: Redis.
Banco de dados de colunas: Nesse tipo de banco de dados, os dados são armazenados em colunas, em vez de linhas como nos bancos de dados SQL. Isso permite uma recuperação eficiente de dados em grandes conjuntos de informações. Exemplo: Cassandra.
Banco de dados de grafos: Nesse tipo de banco de dados, os dados são armazenados em forma de grafos, com nós representando entidades e arestas representando relacionamentos entre essas entidades. Isso permite consultas eficientes em relacionamentos complexos. Exemplo: Neo4j.

3 - MongoDB é uma das ferramentas NoSQL mencionadas. Suas características principais são:

Banco de dados orientado a documentos que armazena dados em formato JSON-like (BSON).
Flexibilidade de esquema, permitindo documentos com estruturas diferentes.
Suporte a consultas avançadas, índices e agregações.
Escalabilidade horizontal e replicação automática.
Casos de uso típicos incluem armazenamento de dados semi-estruturados, aplicativos web, registros de eventos, gerenciamento de conteúdo e análise de dados.

4 - As diferenças entre a estrutura de dados e o modelo de consulta em bancos de dados SQL e NoSQL são:

Estrutura de dados: Nos bancos de dados SQL, os dados são estruturados em tabelas com esquemas rígidos e relacionamentos definidos. Já nos bancos de dados NoSQL, a estrutura de dados é flexível, permitindo a adição de campos e alterações de esquema sem afetar os outros documentos.

Modelo de consulta: Nos bancos de dados SQL, as consultas são feitas usando a linguagem SQL, que é padronizada e otimizada para consultas relacionais, como seleção, projeção, junção e agregação. Nos bancos de dados NoSQL, o modelo de consulta varia de acordo com o tipo de banco de dados. Cada tipo possui sua própria abordagem para recuperar dados, como consultas baseadas em documentos, consultas de chave-valor, consultas de grafos ou consultas de colunas amplamente distribuídas.

A escolha entre um banco de dados SQL e NoSQL depende dos requisitos do projeto. Bancos de dados SQL são adequados para aplicativos com esquemas de dados bem definidos e que exigem transações complexas. Bancos de dados NoSQL são mais adequados para casos em que a flexibilidade do esquema, escalabilidade e desempenho são prioridades, como armazenamento de dados não estruturados, aplicativos web de alto tráfego e análise de big data.

5 - A estrutura de um objeto JSON (JavaScript Object Notation) é baseada em pares de chave-valor, em que os valores podem ser de diferentes tipos, como números, strings, booleanos, arrays ou objetos aninhados. Em bancos de dados NoSQL orientados a documentos, como o MongoDB, o JSON é amplamente usado para armazenar informações complexas.

Um exemplo de objeto JSON em um banco de dados NoSQL orientado a documentos é o seguinte:

{
    "nome": "João",
    "idade": 30,
    "endereço": {
      "rua": "Rua Principal",
      "cidade": "São Paulo",
      "país": "Brasil"
    },
    "hobbies": ["esportes", "música", "leitura"]
  }

Esse objeto JSON pode ser armazenado em um banco de dados NoSQL orientado a documentos e facilmente recuperado para manipulação ou consulta. A estrutura flexível do JSON permite adicionar ou remover campos conforme necessário, tornando-o adequado para armazenar informações complexas com diferentes níveis de aninhamento e variedade de tipos de dados.
